# 最新 PRD 差距修复计划

基准文档：`docs/prd.md`
状态说明：`[x]` 已完成，`[ ]` 待实施。

## P0 数据与安全

- [x] 任务主状态统一为 `pending`、`parent_review`、`confirmed`、`needs_resubmit`，迁移历史 `submitted`、`ai_checking`。
- [x] 任务积分默认值改为 1，允许并展示 0 分。
- [x] 新增可选任务备注，移除任务公开接口、类型和 UI 中的 AI 选项。
- [x] 编辑限制为 `pending`、`needs_resubmit`；删除限制为无提交记录的 `pending`。
- [x] 提交、图片记录和任务状态更新使用同一事务。
- [x] 家长审核、状态更新和积分发放使用同一事务及行锁。
- [x] 同一提交只允许一条审核；重复确认返回已有审核和积分结果。
- [x] 要求补充、心愿驳回和兑换拒绝强制填写原因。
- [x] 新增兑换申请审计记录；预扣和退款流水关联同一请求 ID。
- [x] 家长不可删除已兑换心愿；仅孩子可删除自己的 `rejected` 心愿。
- [x] 图片读取要求登录并校验家庭、孩子归属；前端使用 Bearer token 加载。

## P1 产品完整性

- [x] 新增 `GET /family/context`，创建任务和家长心愿页使用当前家庭唯一孩子。
- [x] 任务详情返回全部提交、图片、备注和对应审核，保留最新 `submission` 兼容字段。
- [x] 家长看板展示孩子积分、待确认任务的备注、缩略图和积分。
- [x] 孩子心愿展示当前积分、所需积分、差额、进度和完整积分流水。
- [x] 兑换拒绝后向孩子展示退款金额和家长原因。
- [x] 按 `Asia/Shanghai` 日期和截止时间计算 `isOverdue`。
- [x] 打卡页支持本地预览、单图失败提示和单图重试，支持无照片任务并限制最多 9 张。
- [x] 全新数据库只创建当前 MVP 表；旧 AI、错题、周报表仅在已有数据库中保留，不进入运行时。

## P1 孩子端心愿兑换奖励反馈

- [x] 新增独立的 `WishRedeemCelebration` 客户端组件，仅接收显示状态和音效设置，不把展示逻辑放进页面路由。
- [x] 仅在兑换申请接口成功且页面完成心愿、积分和流水状态更新后触发；失败、积分不足、重复操作、刷新页面和家长确认兑现均不触发。
- [x] 展示约 2.2 秒的轻量礼盒星光特效，使用符合动物岛风格的透明 WebP 奖励插画、柔光和星光效果，不使用白色信息卡或长时间全屏领奖仪式。
- [x] 特效层不展示心愿名称、扣除积分、剩余积分或状态说明文案；业务状态由心愿卡标签表达。
- [x] 支持点击背景或按 `Escape` 提前关闭；保留仅供读屏使用的 `role="status"` 成功播报。
- [x] 使用 Web Audio API 合成约 0.8 秒的明亮三音提示，不引入音频依赖或外部音频文件；播放失败不得影响兑换结果。
- [x] 页面提供默认开启的音效开关，使用 `localStorage` 保存选择；静音后刷新仍保持，存储不可用时安全降级。
- [x] 支持 `prefers-reduced-motion`，减少动态效果时仅保留淡入淡出和静态成功信息，并清理定时器与音频上下文。
- [x] 复用现有服务端“兑换申请成功”产品事件作为奖励反馈展示的代理指标，不新增后端接口、数据库字段或兑换状态。
- [x] 增加 Playwright 心愿兑换行为覆盖，验证单次触发、静音持久化、减少动态效果、键盘关闭及 390px 布局；失败分支继续由仅在成功响应后设置庆祝状态的实现边界保证。
- [x] 完成后运行前端类型检查、构建和相关心愿兑换 E2E，并同步更新 `docs/implement.md` 与 `docs/documentation.md`。

## 默认约定

- 继续使用服务端本地图片存储和浏览器 `localStorage` 会话令牌。
- 当前不接入 AI、对象存储、多孩子、任务模板或周报。
- 数据迁移保持向后兼容，不删除已有业务记录和历史扩展表。
- 心愿奖励反馈只表示兑换申请成功，不改变“兑换申请中 → 家长确认 → 已兑换”的业务状态流转。
- 本轮奖励反馈不修改后端 API、数据库、积分事务或家长确认流程。
