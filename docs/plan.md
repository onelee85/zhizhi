# MVP 开发计划

本文档根据 `docs/prd.md` 整理，用于指导第一版 MVP 研发。功能范围以 PRD 为准，不扩展 PRD 未定义的功能。

## 目标

第一版验证家庭学习管理最小闭环：

```text
家长布置任务 -> 孩子查看任务 -> 孩子拍照打卡 -> AI 辅助检查 -> 家长确认 -> 每周复盘
```

MVP 优先保证任务打卡闭环稳定，再逐步加入 AI 检查、错题沉淀、薄弱点分析和周报。

## 阶段一：任务打卡闭环

目标：跑通家长布置任务、孩子查看任务、孩子上传打卡、家长查看结果的基础闭环。

功能范围：

- 家长使用用户名和密码登录。
- 孩子使用用户名和密码登录。
- 家长创建每日学习任务。
- 孩子查看今日任务清单和任务详情。
- 孩子勾选完成并上传 1 到 9 张图片。
- 家长在今日看板查看任务状态、上传图片和提交结果。
- 家长确认通过或要求补充。

关键数据：

- `user`
- `family_member`
- `study_task`
- `task_submission`
- `submission_image`
- `parent_review`

交付标准：

- 一个家庭可以正常使用。
- 每天可以创建、完成、查看任务。
- 任务状态流转完整。
- 图片上传稳定可用。
- 所有数据库写入必须校验家庭权限。

验证方式：

- Service 层单测覆盖任务创建、提交打卡、家长审核和权限校验。
- API 集成测试覆盖创建任务、查询今日任务、提交打卡、家长确认和要求补充。
- E2E 测试覆盖家长创建任务到孩子提交、家长查看并确认的完整链路。

## 阶段二：AI 完成度检查

目标：在打卡闭环中加入 AI 检查能力，减少家长人工检查压力。

功能范围：

- 孩子提交后自动触发 AI 检查。
- AI 判断上传内容是否匹配任务。
- AI 判断完成度。
- AI 输出异常提示、简短结论和建议动作。
- 家长查看 AI 检查结论。
- 家长可覆盖 AI 判断，标记 AI 判断不准确。

关键数据：

- `ai_check_result`
- `task_submission`
- `parent_review`

交付标准：

- AI 检查结果可展示。
- AI 无法判断时明确提示，不能强行给结论。
- AI 失败不影响基础打卡流程和原图查看。
- AI 原始返回必须存入 `raw_result`。

验证方式：

- Service 层单测覆盖 AI 输出解析、失败兜底、状态更新。
- API 集成测试覆盖提交后触发 AI、查询 AI 结果、家长覆盖 AI 判断。
- 使用模拟 AI 返回验证完成、部分完成、疑似未完成、无法判断和检查失败。

## 阶段三：错题记录

目标：沉淀学习问题，为薄弱点分析和周报提供数据基础。

功能范围：

- 家长从任务详情中手动添加错题。
- AI 生成疑似错题记录，等待家长确认。
- 错题列表按科目查看。
- 错题关联原始任务、提交和图片。
- 错题可标记掌握状态。

关键数据：

- `wrong_question`
- `ai_check_result`
- `study_task`
- `submission_image`

交付标准：

- 错题可添加、查看、按科目筛选。
- 错题可关联到原始任务和原始图片。
- 错题能进入周报统计。

验证方式：

- Service 层单测覆盖手动添加、AI 候选入库、掌握状态更新。
- API 集成测试覆盖错题创建、列表查询和权限隔离。
- E2E 测试覆盖家长从任务详情添加错题并在错题列表查看。

## 阶段四：薄弱点分析和周报

目标：让家长每周获得可执行的学习复盘和下周建议。

功能范围：

- 汇总一周任务统计。
- 汇总按科目统计。
- 汇总错题统计。
- 汇总 AI 异常记录和家长确认记录。
- 生成薄弱点分析。
- AI 生成每周学习报告。
- 家长查看历史周报。

关键数据：

- `weekly_report`
- `study_task`
- `task_submission`
- `ai_check_result`
- `wrong_question`
- `parent_review`

交付标准：

- 每周可以生成一份结构化报告。
- 报告包含完成率、错题、薄弱点和下周建议。
- 报告内容与实际数据相关，不能泛泛而谈。
- 周报生成失败支持重新生成。

验证方式：

- Service 层单测覆盖统计口径、周区间、空数据、失败重试。
- API 集成测试覆盖生成周报、查询历史周报。
- 使用构造的一周数据验证报告内容与任务、错题、异常数据一致。

## 技术与质量要求

- 使用 Next.js App Router、TypeScript、MySQL、Qiniu Cloud Storage、Alibaba Bailian、Tailwind CSS、shadcn/ui、GitHub Actions。
- 登录使用 MySQL 应用用户体系，家长和孩子均使用用户名 + 密码。
- `app/` 只放路由、布局、页面和 route handlers。
- 业务逻辑放在 `features/`，服务端基础设施放在 `server/`。
- 请求参数使用 Zod 校验。
- 业务模块使用 service + repository pattern。
- 不在客户端暴露数据库账号密码、Qiniu Secret、Bailian API Key。
- AI 调用异步执行，不阻塞孩子提交。

## MVP 总体验收

- 家长可以创建每日任务。
- 孩子可以查看任务并上传打卡。
- 家长可以查看今日完成情况。
- AI 可以生成完成度检查结果。
- 家长可以确认或要求补充。
- 系统可以基于一周数据生成报告。
