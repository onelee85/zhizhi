# MVP 开发计划

本文档根据 `docs/prd.md` 整理，用于指导第一版 MVP 研发。功能范围以 PRD 为准，不扩展 PRD 未定义的功能。

## 目标

第一版验证家庭学习管理最小闭环：

```text
家长布置任务 -> 孩子查看任务 -> 孩子拍照打卡 -> AI 辅助检查 -> 家长确认 -> 发放积分 -> 兑换心愿 -> 每周复盘
```

MVP 优先保证任务打卡闭环稳定，再逐步加入积分愿望激励、日历面板、AI 检查、错题沉淀、薄弱点分析和周报。

## 阶段一：任务打卡闭环

目标：跑通家长布置任务、孩子查看任务、孩子上传打卡、家长查看结果的基础闭环。

功能范围：

- 家长使用用户名和密码登录。
- 孩子使用用户名和密码登录。
- 家长创建每日学习任务。
- 孩子查看今日任务清单和任务详情。
- 孩子勾选完成并上传 1 到 9 张图片。
- 家长在今日看板查看任务状态、上传图片和提交结果。
- 家长确认通过或要求补充。

关键数据：

- `user`
- `family_member`
- `study_task`
- `task_submission`
- `submission_image`
- `parent_review`

交付标准：

- 一个家庭可以正常使用。
- 每天可以创建、完成、查看任务。
- 任务状态流转完整。
- 图片上传稳定可用。
- 所有数据库写入必须校验家庭权限。

验证方式：

- Service 层单测覆盖任务创建、提交打卡、家长审核和权限校验。
- API 集成测试覆盖创建任务、查询今日任务、提交打卡、家长确认和要求补充。
- E2E 测试覆盖家长创建任务到孩子提交、家长查看并确认的完整链路。

## 阶段二：积分愿望激励

目标：在打卡闭环中加入积分愿望激励，提升孩子持续完成任务的动力。

功能范围：

- 家长创建或编辑未完成任务时设置任务积分。
- 家长确认任务完成后，系统发放该任务积分。
- 孩子提交愿望标题和说明。
- 家长为愿望设置所需积分并通过，或驳回愿望。
- 孩子积分足够时发起兑换申请。
- 孩子申请兑换时系统立即扣减积分；家长确认后标记愿望已兑换，拒绝时返还积分并恢复为可兑换。
- 积分获得和扣减必须记录流水。
- 孩子可以修改或删除被家长驳回的心愿；修改后状态回到“待家长设置积分”，并清空所需积分、家长审核记录和驳回原因。
- 第一版周报不展示积分、愿望和兑换数据。

关键数据：

- `study_task.reward_points`
- `parent_review`
- `child_point_account`
- `point_ledger`
- `wish`

交付标准：

- 家长确认任务完成后才发放积分；后续接入 AI 后，AI 判断完成也不直接发放积分。
- 同一任务积分只能发放一次。
- 家长不能修改孩子提交的愿望原文。
- 孩子申请兑换时必须原子校验并扣减积分，不能通过多个待确认申请透支余额。
- 只有被家长驳回的心愿可被孩子修改或删除。
- 积分账户和积分流水写入必须校验家庭权限。

验证方式：

- Service 层单测覆盖积分发放幂等、愿望审核、兑换申请、兑换扣减和权限校验。
- API 集成测试覆盖设置任务积分、确认任务发积分、孩子提交愿望、家长设置愿望积分或驳回、孩子申请兑换、家长确认兑换。
- E2E 测试覆盖孩子完成任务获得积分，并用积分发起心愿兑换申请的完整链路。

当前实现状态：

- 后端 API 已实现任务奖励积分、积分账户/流水、愿望提交、愿望审核、兑换申请和家长确认兑换。
- 后端 API 已实现孩子修改/删除被驳回的心愿：新增 `GET /wishes/:wishId`、`PATCH /wishes/:wishId` 和 `DELETE /wishes/:wishId`，仅 `rejected` 状态、仅心愿所有者孩子可调用。
- 前端已接入任务奖励积分、孩子心愿清单、家长愿望管理、积分账户和积分流水展示。
- 前端孩子端在 `rejected` 状态的心愿上展示"修改心愿"和"删除心愿"入口，删除前使用统一确认弹窗；修改页 `/child/wishes/[wishId]/edit` 复用创建表单结构并展示驳回原因。
- 当前家长端愿望管理仍使用 Demo seed 的 `child-1`，待家庭孩子列表接口完成后改为真实孩子选择。
- 服务层单测、API 集成测试和 E2E 测试仍待补齐。

## 阶段三：日历面板

目标：在现有任务系统上新增月视图日历面板，让家长和孩子可以按日期查看任务，并支持家长在日历中创建、编辑和删除未完成任务；同时新增历史任务归档，避免已完成任务长期堆积在普通任务列表。

功能范围：

- 默认展示月视图日历面板。
- 日历顶部显示当前年月。
- 支持切换上个月 / 下个月。
- 支持回到今天。
- 今天在日历中需要有明显标识。
- 每个日期格子展示当天任务摘要。
- 家长可以查看当前家庭内所有孩子的任务。
- 孩子只能查看分配给自己的任务。
- 家长点击某一天可以创建任务，默认任务日期为点击日期。
- 家长点击已有任务可以进入任务详情，并编辑任务。
- 家长可以删除未完成任务，删除前必须二次确认。
- 日历创建的任务必须同步出现在任务列表中，任务列表创建的任务也必须同步出现在日历中。
- 家长确认完成超过 7 天的任务自动进入历史任务，不再展示在普通家长任务列表和孩子任务列表。
- 家长和孩子均有【历史任务】入口；家长查看当前家庭归档任务，孩子只查看自己的归档任务。
- 日历面板仍展示已归档任务，并标记“已归档”。
- 历史任务列表和详情只读，不支持编辑、删除、重新提交或重新审核。
- 移动端需要基础适配，避免月视图任务文本拥挤或遮挡。
- 周视图、日视图、拖拽改日期、重复任务、农历 / 节假日、多日任务、日历订阅、第三方日历同步和孩子自主创建任务不纳入本阶段。

关键数据：

- `study_task`
- `study_task.due_date`
- `study_task.due_time`
- `study_task.reward_points`
- `study_task.status`
- `study_task.child_user_id`
- `study_task.deleted_at`

交付标准：

- 家长可以进入日历面板查看当前月份任务。
- 家长可以切换上个月、下个月。
- 家长可以点击今天按钮回到当前月份。
- 家长可以点击某一天创建任务。
- 创建后的任务正确显示在对应日期。
- 家长可以点击任务进入详情。
- 家长可以编辑任务信息。
- 编辑后日历中的任务信息同步更新。
- 家长可以删除未完成任务。
- 删除后任务不再显示在日历和任务列表中。
- 孩子只能查看自己的任务，不能创建、编辑或删除任务。
- 日历中的任务状态与任务列表保持一致。
- 家长确认完成超过 7 天的任务从普通任务列表隐藏。
- 家长和孩子可以从【历史任务】入口查看归档任务。
- 历史任务详情保留任务、照片、AI 检查、家长确认和积分信息，且只读。
- 积分发放仍然只发生在家长审核通过后。
- 所有日历任务查询和写入必须校验家庭权限。

验证方式：

- Service 层单测覆盖按月份查询任务、按家庭和孩子隔离数据、创建任务默认日期、编辑任务日期、删除未完成任务、归档任务判断和历史任务权限隔离。
- API 集成测试覆盖日历任务查询、历史任务查询、家长按日期创建任务、编辑任务、删除未完成任务和孩子只读权限。
- E2E 测试覆盖家长进入日历面板、切换月份、回到今天、按日期创建任务、编辑任务、删除未完成任务、普通列表隐藏归档任务和历史任务入口查看。
- E2E 测试覆盖孩子进入日历面板并只能查看自己的任务，以及孩子通过历史任务入口查看自己的归档任务。

当前实现状态：

- PRD 已定义日历面板需求。
- 现有任务表已有 `due_date`、`due_time`、`reward_points`、`status`、`child_user_id` 和 `deleted_at`，可支撑日历面板基础能力。
- 后端已新增 `GET /tasks/calendar?month=YYYY-MM` 月视图任务查询接口，使用当前登录用户角色做权限隔离：家长返回家庭整月任务，孩子只返回自己的整月任务。
- 前端已新增 `/parent/calendar` 和 `/child/calendar` 日历面板页面，复用 `TaskCalendarPanel`。
- 日历 UI 已明确为类 Apple Calendar 月视图在 `animal-island-ui` 风格下的落地：桌面端使用 7 列月历 + 右侧选中日期检查器，iPad 使用完整 7 列月历 + 下方选中日期任务抽屉；日期格展示最多 3 条任务摘要，超出显示“还有 N 项”。
- 家长日历支持上月 / 下月、今天、今日高亮、点击日期选中、点击日期 `+` 创建任务、点击任务进入详情、在选中日期检查器编辑或删除未完成任务；创建任务页可接收 `dueDate=YYYY-MM-DD` 作为默认截止日期。
- 孩子日历为只读视图，点击任务按状态进入打卡页或提交结果页。
- 顶部导航、家长任务管理页和孩子今日任务页已加入日历入口。
- Service 层单测、API 集成测试和 E2E 测试仍待补齐。

## 阶段四：AI 完成度检查、错题记录、薄弱点分析和周报

目标：在打卡闭环中加入 AI 检查能力，减少家长人工检查压力，并沉淀学习问题；基于任务、错题和 AI 检查数据生成薄弱点分析和每周学习报告。

功能范围：

- 孩子提交后自动触发 AI 检查。
- AI 判断上传内容是否匹配任务。
- AI 判断完成度。
- AI 输出异常提示、简短结论和建议动作。
- 家长查看 AI 检查结论。
- 家长可覆盖 AI 判断，标记 AI 判断不准确。
- 家长从任务详情中手动添加错题。
- AI 生成疑似错题记录，等待家长确认。
- 错题列表按科目查看。
- 错题关联原始任务、提交和图片。
- 错题可标记掌握状态。
- 汇总一周任务统计。
- 汇总按科目统计。
- 汇总错题统计。
- 汇总 AI 异常记录和家长确认记录。
- 生成薄弱点分析。
- AI 生成每周学习报告。
- 家长查看历史周报。

关键数据：

- `weekly_report`
- `study_task`
- `task_submission`
- `ai_check_result`
- `wrong_question`
- `parent_review`
- `submission_image`

交付标准：

- AI 检查结果可展示。
- AI 无法判断时明确提示，不能强行给结论。
- AI 失败不影响基础打卡流程和原图查看。
- AI 原始返回必须存入 `raw_result`。
- 错题可添加、查看、按科目筛选。
- 错题可关联到原始任务和原始图片。
- 错题能进入周报统计。
- 每周可以生成一份结构化报告。
- 报告包含完成率、错题、薄弱点和下周建议。
- 报告内容与实际数据相关，不能泛泛而谈。
- 周报生成失败支持重新生成。

验证方式：

- Service 层单测覆盖 AI 输出解析、失败兜底、状态更新。
- API 集成测试覆盖提交后触发 AI、查询 AI 结果、家长覆盖 AI 判断。
- 使用模拟 AI 返回验证完成、部分完成、疑似未完成、无法判断和检查失败。
- Service 层单测覆盖手动添加、AI 候选入库、掌握状态更新。
- API 集成测试覆盖错题创建、列表查询和权限隔离。
- E2E 测试覆盖家长从任务详情添加错题并在错题列表查看。
- Service 层单测覆盖统计口径、周区间、空数据、失败重试。
- API 集成测试覆盖生成周报、查询历史周报。
- 使用构造的一周数据验证报告内容与任务、错题、异常数据一致。

## 技术与质量要求

- 使用 Next.js App Router、TypeScript、MySQL、Qiniu Cloud Storage、Alibaba Bailian、Tailwind CSS、shadcn/ui、GitHub Actions。
- 登录使用 MySQL 应用用户体系，家长和孩子均使用用户名 + 密码。
- `app/` 只放路由、布局、页面和 route handlers。
- 业务逻辑放在 `features/`，服务端基础设施放在 `server/`。
- 请求参数使用 Zod 校验。
- 业务模块使用 service + repository pattern。
- 不在客户端暴露数据库账号密码、Qiniu Secret、Bailian API Key。
- AI 调用异步执行，不阻塞孩子提交。

## MVP 总体验收

- 家长可以创建每日任务。
- 孩子可以查看任务并上传打卡。
- 家长可以查看今日完成情况。
- AI 可以生成完成度检查结果。
- 家长可以确认或要求补充。
- 家长确认任务后系统可以发放任务积分。
- 孩子可以提交愿望并发起兑换申请。
- 家长可以设置愿望积分、驳回愿望或确认兑换。
- 家长和孩子可以通过日历面板按日期查看任务。
- 家长可以在日历面板中按日期创建、编辑和删除未完成任务。
- 日历面板与任务列表、任务详情的数据保持一致。
- 系统可以基于一周数据生成报告。
